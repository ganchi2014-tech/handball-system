> 出典：IHF ルール 2025・EHF教材・DHB アスレティクコンツェプト（運動能力育成構想）・ハンドボール エボリューション（進化）・KNSU・VIBSS・PubMed査読論文

## 【位置づけ】なぜバックプレーヤーの個人DFが必要か

```
バックプレーヤー（RL/RM/RR = Rückraumspieler / arrière）は
攻撃では最も目立つポジション。
守備では「中央ブロック（ゴール前中央を固める守備の中心部分。独：Mittelblock）」の一部を担い、
フォーメーション守備の枠組みの中で相手バックとの1on1（1対1の競り合い。独：Zweikampf）が発生する。

6:0・5:1のどちらの陣形でも、
相手バックが突破を試みたときは最終的に個人技術で止めるしかない。
フォーメーションを知っていても、個人DFができなければ意味がない。
```

## 【A】基本姿勢（グルントシュテルング（基本姿勢） / ディフェンシブ Stance / Postura defensiva básica）

### A-1. 足・重心・腕

**ドイツ語圏の標準記述（DHB / KNSU / VIBSS）：**

```
足の位置（バインシュテルング（脚位置））：
  軽く開いた状態（leichte Grätschstellung）
  両足に均等に体重をかける
  斜め足配置（diagonale バインシュテルング（脚位置））も使用

膝（Knie）：
  軽く曲げる（leicht gebeugt）
  絶対に伸ばしきらない

重心（Körperschwerpunkt）：
  低く保つ
  足の拇指球（足の親指の付け根のふくらみ。独：Ballen）に乗る — 踵（かかと）は浮かせ気味に
  重心が高いと方向転換が遅れる

上体（Oberkörper）：
  やや前傾
  肩を膝の前方に出す意識

腕（Arme）：
  直角に曲げ（im rechten Winkel）、体側に開く
  手のひらを前方に向ける（Handflächen nach vorne）
  常に相手と接触できる中程度の高さ（halbhohe Position）

視線：
  相手の腰・へそ付近を見る
  足先・目線・上体の動きに釣られない
```

**英語圏の記述（IHF Teaching ハンドボール / ハンドボール エボリューション（進化））：**
```
balls of the feet（拇指球に乗る）
knees slightly bent（膝を軽く曲げる）
arms open, wide, and high（腕は広く・高く・開いて保つ）
small, fast steps（小さく素早いステップを踏める状態で待つ）
```

## 【B】フットワーク（Beinarbeit / Footwork / Trabajo de piernas）

### B-1. 基本原則

```
① つま先・拇指球立ちで動く（auf den Zehenspitzen）
  → 踵をつけて構えると、つま先立ちのときに比べ方向転換の動き出しが0.2〜0.3秒ほど遅れるとされる（目安）

② 重心を低く保ちながら、地面に近い平らなステップで動く
  → ジャンプしない（"Never jump" — 英語圏コーチング共通の注意事項）
  → 飛び上がると相手に抜かれる、ファウルを取られやすい

③ 相手から背中を向けない
  → 後退時も体の向きは正面を保つ
```

### B-2. ステップ技術3種（ドイツ語圏用語）

**ナハシュテルシュリット（サイドステップ）（ナッハシュテルシュリット / 追従ステップ）**
```
横方向移動の基本
前の足を出してから後の足を引き寄せる
重心を常に2足間に保つ
→ 小さい移動・密着マーキングに使う
```

**クロイツシュリット（クロスステップ）（クロイツシュリット / クロスオーバーステップ）**
```
大きな横移動が必要な場面に使用
足を交差させて大きく素早くカバー
ハンドボール エボリューション（進化）：「クロスステップは三角形DF移動の生体力学的キーエレメント」
→ ただし姿勢と方向を保つこと（体が流れないよう注意）
```

**アウスファルシュリット（踏み込みステップ）（アウスファルシュリット / ランジステップ）**
```
前方・斜め方向への急接近時に使用
相手がドライブを仕掛けてきたとき、一歩踏み込んで道を塞ぐ
```

### B-3. トレーニング方法（ハンドボール エボリューション（進化）推奨）

```
「手を後ろで組んで1on1練習」：
  腕が使えない状態で守ることを強制する
  → 正しいフットワークを身につけざるを得ない環境を作る
  → 相手のジャージを引っ張る反則習慣を排除できる
```

## 【C】合法的な身体接触（Erlaubter Körperkontakt / Legal ボディ Contact）

### C-1. IHFルール8（Rule 8 / Regel 8）に基づく規定

**許可されている行為（erlaubt / permitted）：**
```
① ボディーブロック（Körperstellen）
   体幹で相手の進路を塞ぐこと
   → 腕ではなく体幹（胴体）を使うことが重要

② 曲げた腕での追従接触（Begleiten mit angewinkelten Armen）
   曲げた腕を使って相手と身体接触しながらマーキングを継続する
   → 「押す」ではなく「寄り添う」
   → 相手の動きを感じながら位置を把握する

③ 手を体に添わせて追う（Accompanying contact）
   手を相手の体に添わせながら動きを追う
   → 掴む・押すは反則

④ ボールをオープンハンドで払い落とす
   開いた手（open hand）でボールを相手の手から払い落とすこと
   → 任意の方向から可
```

**禁止されている行為（verboten / prohibited）：**
```
× 腕・手・脚でブロック（腕で壁を作る）
× 相手を掴む・引っ張る・押す
× 肘を危険に使う
× 相手に飛びかかる
```

**実践的解釈（handballregeln.de / owayo.de）：**
```
「合法：ボールを狙う動作、および腕でなく体幹でのブロック」
「体で相手をブロックできるが、拘束してはならない」
「手を体に添わせて追うことは可。押すことは不可」
```

## 【D】ドライブ（突破）への対応（Durchbruchverteidigung）

### D-1. 基本原則：ボール-Mann-Tor

```
「ボール → 相手 → ゴール」の直線上に常に位置する原則
→ ディフェンダーは相手とゴールの間に入り続ける
→ 相手がゴールに近いほど、より密着したマーキングが必要
```

### D-2. 対応技術4つ

**① 斜め前方から接近（Diagonale Annäherung）**
```
相手の正面からではなく、シュートを打つ利き手の側（独：Wurfarmseite＝投げ腕側）から斜めに接近
→ 利き手シュートコースを塞ぎながら、逆方向（外側）へ誘導
→ 内側へのドライブを遮断し、難しい角度のシュートへ追い込む
```

**② 踏み出し（Aussteppen / Step Out）**
```
相手がドライブを仕掛けてくる前に、前方に踏み出して間合いを詰める
→ 早すぎるフェイントへの警戒：腰を見て重心が本当に動くまで踏み出さない
```

**③ 重心で対応（腰を読む）**
```
相手の腰・へそを見て重心移動を読む
足先・上体の動き・視線には釣られない
→ 重心が動いた方向が「本当に行きたい方向」
```

**④ アンザウゲン（引き寄せ）（アンザウゲン / 引き込み）**
```
わざと相手に進めそうなスペースを見せて誘い込むようなポジションを取り、
相手がドライブを仕掛けてきたところで体で詰める技術
→ 相手の動きを先に読む力（先読み）が高いレベルで必要。経験者向け。
→ 相手を誘き出してカウンターアタックを誘発させる戦術的DF
```

**ルール上の注意：**
```
体幹でのブロック（Körperstellen）は合法
腕・肘でのブロック、掴む・押すは反則
ジャンプしてのブロックは「着地時に相手にぶつかる」形になり反則を取られやすい
→ ジャンプせずに体幹で対応が原則
```

## 【E】先読み・アンティシペーション（アンティツィパツィオン（先読み） / アンティシペーション（先読み））

### E-1. 科学的根拠

```
研究（Seidel et al., PubMed 2020）：
  ハンドボールの専門家選手は相手の身体運動（kinematic information）から
  早期に情報を取得し、ボールの軌道を初心者より精度高く・速く予測できる
  エキスパートと初心者では「読む動作シーケンスの種類」が異なる
```

### E-2. 読み取るべき手がかり（アンティシペーション（先読み） Cues）

```
① 腰・重心の移動（最重要）
   → ドライブ方向を先行して示す最も信頼できる情報

② 肩の向き（肩軸）
   → ドライブ方向・シュートの狙いを示す

③ 体重移動の方向
   → どちらの足に重心が乗っているか

④ ドリブルのリズム変化
   → 加速・減速のパターン変化はドライブや止まりのサイン

⑤ 目線
   → 相手がどこを見ているか（ただし経験者は意図的に視線を外す）

⑥ チームのセットプレーパターン
   → 相手のクロスラン（クロイツ（クロス）lauf）・コンビネーションを事前に把握しておく
```

**ヴァニャ・ラディッチ（フィンランドハンドボールGKコーチ）：**
```
「良いポジションがあってこそ、相手の動きを予測し反応できる」
→ アンティシペーションはポジション取りと密接に連動する
```

## 【F】スクリーン（シュペレ（妨害））への対応

### F-1. スクリーンの種類（シュペレン（妨害）arten）

```
Vordersperre（フォルダーシュペレ）：前方から塞ぐ
Seitensperre（ザイテンシュペレ）：横から塞ぐ
Hintersperre（ヒンターシュペレ）：背後から塞ぐ
```

### F-2. 対応技術4つ

**① 突き破る（Drüberlaufen / Push Through）**
```
スクリーンが小さい、または到達前に回避できる場合
密着守備（Manndeckung）では困難
```

**② 迂回（Umweg / Go Around）**
```
スクリーンの外側を迂回する
→ 相手シューターとの距離が開くリスクあり
→ シュートラインが遠い場合に有効
```

**③ スイッチ（アプガーベ（パス放出） / Switch）**
```
担当を隣の味方と入れ替える
バックDF同士、またはバックDFとセンターDFのスイッチ
→ 声での合図（「Übergabe!」「Ich hab!」）が必須
→ コミュニケーション不足でマークが空くと即失点
```

**④ ヘッジ（Helfen / Hedge）**
```
スクリーナー担当のDFが一歩出てドライブコースを塞ぎ、
本来のマーカーが戻れるよう時間を作る
→ 中間的な対応。味方が追いつくまでの時間稼ぎ
```

**EHF教材の原則：**
```
「取り次ぎ（Übergabe）と引き受け（Übernahme）の手順を全員が身につけておく」
「誰かが常に担当している状態を維持する」
「クライスレーファー（ピボット）には常にマーカーを付けておく」
```

## 【G】攻守切り替え（Umschaltverhalten / Transition Defense）

### G-1. バックプレーヤー固有の問題

```
バックプレーヤーは攻撃時にゴールから最も遠い位置（ハーフ付近）で活動。
ボールロスト時に守備への切り替えが全ポジション中最も遅れやすい。
```

### G-2. 切り替え原則5つ

**① 即時ダッシュバック（Sprint Back）**
```
ボールロスト直後にスプリントでゴール方向に戻る
→ サイドステップで戻るより向きを変えてダッシュする方が速い
→ 格好より速さを優先
```

**② 3秒の原則（3-Sekunden-Prinzip）**
```
ボールロスト後3秒以内に守備形勢を立て直すことが目標
→ 速攻の波に対応するために最速での帰還が必須
```

**③ 優先カバー順**
```
1. ゴール中央（センター）を守るラインに入ること
2. 次に自分の担当エリア（守備ゾーン）に戻る
3. ウィングの速攻はGKと残ったDFが対応することが多い
```

**④ 事前の守備準備（アンティツィパツィオン（先読み） des ボールverlustes）**
```
攻撃中から相手のカウンターを予想し、
常に「今ボールを失ったら」を意識した立ち位置を取る
→ 攻撃参加しながらもリスク管理を忘れない
```

**⑤ ポジション回復の優先順位**
```
抜かれた・崩された後：
  相手を追いかけるのではなく「相手とゴールの間に戻る」を最優先
  → ボールを追わず、ゴール前に帰還
  → 声でカバーリングを要請しながら戻る
```

## 【H】守備コミュニケーション（Abspracheverhalten）

### H-1. 基本コマンド構造

```
「名前 + 動作 + トリガー」
例：「マルティン、出ろ、バック来てるぞ！」
   「Martin, rausgehen, Rückraumspieler!」
```

### H-2. 主要コマンド一覧

| ドイツ語 | 日本語 | 使う場面 |
|---|---|---|
| `Ich hab!` | 俺が持つ！ | 担当を引き受けるとき |
| `Hilf!` | 助けてくれ！ | 抜かれそうなとき |

  → 前に出てプレッシャーをかける

  → 3人が三角形を作って動き合う「三角形の協力」（独：Dreiecksbewegung）で互いをカバー

## じゃんけん構造（個人DF）

```
DF前に出る（ヘラウスリュッケン（前飛び出し））
  → OF：フェイントで抜く → DF：読んでいれば対応できる
  → OF：パスを出す    → DF：次のOFがオープンになる

DF引く（引き込み）
  → OF：シュートを打てる距離が近い → DF：ブロックが効く
  → OF：ドライブで内側へ      → DF：体幹ブロック合法

DF側面から接近（斜め）
  → OF：内側は塞がれている → 外側へ誘導
  → OF：外側シュートを打つ → GKが角度を読んでいれば対応可

→ 絶対最強のDF技術はない。相手の利き手・スタイル・試合状況で最適解が変わる
```

## 【R】6人DFの集合知メカニズム（なぜ声なしでは機能しないか）

> **科学的背景**：ハンドボールのDF6人が「一つの思考システム」として機能するには、個人の視野・認知の限界を補う仕組みが必要である。以下は認知科学・スポーツ科学の知見をハンドボールDF文脈に適用した解説。

---

### R-1. 個人視野の限界と集合補完

#### 人間の視野の構造

```
視野の全体像（生物学的制約）：
  水平視野の総範囲：約180〜200°（両目合わせた時）
  高解像度（中心窩）：中心約2°程度のみ
  有効注意視野（はっきり注意を向けて見える範囲。"useful field of view"）：中心から左右30〜40°程度（おおむね自分の正面まわりだけ）
  → それ以外はぼんやりした周辺視野（Peripheres Sehen）

実戦での意味：
  相手バックを見ている間、
  隣のDF担当エリア・スクリーンを掛けてくる選手・ポストの動き
  → これらは周辺視野または完全な死角（Toter Winkel）に入る
```

#### DF6人それぞれの死角

```
【スクリーン（Sperre）対応中】
  担当相手と競り合っている最中、背後や側面への動きは見えない
  → スクリーナーの接近を「感じる前に知る」には声しかない

【マーキング中（密着）】
  相手に正対しているとき、後方からの第3の選手の動きは死角
  → 「クライス（ピボット）が右に動いた！」という情報は
    見ていない選手からしか届かない

【前出し（Herausgehen）時】
  自分が前に飛び出すとき、後方のスペースがどう空くか自分では見えない
  → 「前出るな！後ろに穴空く！」の判断は後方のDFにしかできない

DFの視野限界を補完する唯一の手段：声（Kommunikation）
```

#### 研究の裏付け ⚠️確認推奨

```
PubMed / ResearchGate (Seidel et al., 2020 関連)：
  ハンドボールの専門家は「グローバルに分散した知覚手がかり」
  （globally distributed cues）を使って判断する
  → 単一の視覚的焦点ではなく、複数の情報を並列処理
  → チームDF場面では、この分散処理を個人の目だけでは賄えない
```

### R-2. 分散認知（Distributed Cognition / Verteilte Kognition）

#### 概念の定義

```
分散認知（Distributed Cognition）：
  認知プロセスが一人の頭の中ではなく、
  複数の人間・道具・環境にまたがって行われるという理論
  → チーム全体が「一つの認知システム」として機能する

出典：Hutchins (1995) "Cognition in the Wild"
スポーツへの応用：
  Distributed Cognition in Sports Teams (ResearchGate, 2014)
  → チームが成功するためには認知の分散が必要と論じる
```

#### ハンドボールDF 6人の分散認知モデル

```
各ポジションの「専門的な認知担当領域」：

  【バック左（RL）】
    → 左ウィング・左ポスト側の動きを最も正確に把握

  【センター左（HHL）】
    → 左バックのドライブ・スクリーン動向を把握

  【センター中央（HHM）】
    → 全体俯瞰・ポジション最適化のハブ担当

  【センター右（HHR）】
    → 右バックのドライブ・スクリーン動向を把握

  【バック右（RR）】
    → 右ウィング・右ポスト側の動きを最も正確に把握

  【ポスト担当（Kreisbewacher）】
    → ゴール前ピボットの動き・身体位置を最も正確に把握

合計：6人で360°の認知カバレッジが生まれる
→ 声がなければ、各自が独立した孤立システムとなり連携不可
```

#### 実例：声で成立するDF動作

| 場面 | 必要な情報 | 誰から誰へ声が来るか |
|---|---|---|
| マーク交換（Übergabe） | 「誰が誰を持つか」の確認 | スクリーンされた選手 → 隣のDF |
| ローテーション指示 | 「どこに穴が開くか」の予告 | 後方のDF → 前に出るDF |
| スクリーン警告 | 「スクリーンが来る方向・タイミング」 | スクリーナーを見ている別のDF |
| ポスト位置通知 | 「ピボットが右に動いた」 | ポスト担当 → バックDF |
| 前出し制止 | 「今は前に出るな」 | 後方のDF → 前のDF |

```
→ これらはすべて「自分では見えていない情報」の補完である
→ 声がなければ、自分の視野範囲のみの判断に限定される
```

### R-3. トランザクティブ・メモリー（Transactive Memory / Transaktives Gedächtnis）

#### 概念の定義

```
トランザクティブ・メモリー（TMS：Transactive Memory System）：
  グループ内で「誰が何を知っているか・何を担当するか」を
  メンバー全員が暗黙的に把握している分業知識システム
  
  提唱者：Wegner (1987) "Transactive Memory: A Contemporary Analysis"
  スポーツ応用：Leo et al. (2018) スポーツ版TMS尺度の開発・検証
  （Adaptation and validation of the TMSS-S, Sage Journals）

TMSの3要素（Leo et al.より）：
  ① 専門性（Specialization）：誰がどの役割を担当するか
  ② 協調（Coordination）：明示的な計画なしに動ける度合い
  ③ 信頼性（Credibility）：他のメンバーの専門知識への信頼
```

#### ハンドボールDFにおけるTMS

```
長期チームのTMS（成熟期）：
  「ヤマダが右から来たらタナカが中央に寄ってカバーする（＝絞る）」
  → 声に出さなくても体が動く状態
  → Coordinationが高い：明示的な合図なしでローテーションが回る
  → これが「暗黙知」の一形態

急造チームのTMS（未熟期）：
  「誰が誰を担当するのか全員がわかっていない」
  → 同じ相手を2人でマークしてしまう（＝1人の相手に守備が重なる。独：Doppelmarkierung）
  → 誰もマークしないノーマーク（Freilaufen）が多発
  → ローテーション崩壊

TMS研究の知見（Olabisi & Lewis, 2018 Sage Journals）：
  強いTMSを持つチームは明示的な調整努力（声の量）が
  少なくても高いパフォーマンスを発揮できる
  → 逆に言えば：TMS未発達チームは声でカバーするしかない
```

#### 長期チームと急造チームで守備崩壊が起きやすい理由の違い

```
【急造チーム（経験不足）での崩壊原因】
  → 役割分担が不明確（Specialization未確立）
  → 「誰が前に出るか」「誰がカバーするか」に迷いが生じる
  → 声を出す習慣もなく、声の意味も共有できていない
  → 解決策：声のコマンドの明示的な訓練（コマンドの標準化）

【長期チームでの崩壊原因（異なるメカニズム）】
  → TMS依存による「過信・慢心」（声を出さなくても大丈夫という錯覚）
  → 相手の戦術変化にTMSが追いつかない（相手が変えてきたのに古い分業を維持）
  → キーメンバー不在時（怪我・交代）にTMSが崩壊しやすい
  → 解決策：定期的なコミュニケーション再確認・声の習慣維持
```

### R-4. 声の3要素：「誰が・何を・いつ言うか」の標準化

#### なぜ標準化が必要か

```
「声があれば良い」では不十分：
  → 情報が遅すぎる（シュートを打たれてから言っても無意味）
  → 情報が不正確（「来てる」だけでは方向・種類がわからない）
  → 受け手が誰かわからない（全員に向けて言うと誰も反応しない）

→ 「誰が」「何を」「いつ」言うかの設計が必要
```

#### 国別アプローチ ⚠️確認推奨

```
【ノルウェー（Forsvarsskolen）】
  → シンプルなコードワード重視
  → "Bytt"（交代）、"Hjelp"（ヘルプ）、"Press"（前出し）
  → 短い・明確・全員統一
  → 反射的に声が出る訓練（Responsivitet：応答性）を強調
  ⚠️確認推奨：ノルウェー協会公式文書での具体的コマンドリスト要確認

【フランス（FFHB / Omeyer式）】
  → 視覚的サインとの組み合わせ（コード語＋手のサイン）
  → GKが守備全体の指揮者として情報を統括・発信
  → GKの「声」をシステム全体の中枢として設計
  ⚠️確認推奨：FFHB公式育成資料での詳細要確認

【スペイン（Real Federación Española）】
  → Dujshebaev由来の「スキャン→識別→コミット」プロセスの中に
    チームへの情報発信（"comunicar"）を組み込む
  → 情報伝達はプレー判断のプロセスに組み込まれており、
    「声を出すこと」自体がプレーの一部として教育される
  ⚠️確認推奨：スペイン連盟公式文書での詳細要確認

共通原則（3国横断）：
  声は「思い出してから出す」ではなく
  「自動的・反射的に出る」レベルまで訓練する
```

#### 「誰が・何を・いつ」設計表

| 役割 | 何を言うか | いつ言うか |
|---|---|---|
| スクリーナーを見ているDF | 「シュペレ（スクリーン）来るぞ！右から！」 | スクリーンが始まる0.5〜1秒前 |
| ポスト担当DF | 「クライス（ピボット）右！」 | ピボットが動き始めた瞬間 |
| 後方のDF（カバー役） | 「出るな！」or「行け！」 | 前DFが前出しを判断する前 |
| マーク交換する側 | 「ウェルゲーベ（渡すぞ）！」 | スクリーンが完成する直前 |
| マーク交換受ける側 | 「イッヒ・ハブ（俺が持つ）！」 | 渡しの声を受けた直後・即座に |

```
→ 遅い声は情報でなく「事後報告」になる
→ 事後報告はDF崩壊を報告するだけで止められない
```

### R-5. 例外：声なしでも成立するケース

#### 長期コンビネーションの暗黙知（Tacit Knowledge）

```
暗黙知（Tacit Knowledge）の定義：
  「言語化されていないが身体・経験に刻まれた知識」
  → Polanyi (1966) "The Tacit Dimension"

DF場面での暗黙知の具体例：
  ① 長年のコンビが特定のサインプレーに対して
     声なしで自動的にローテーションを実行できる
  ② 相手の同じサインプレーを何十回も見ているため、
     「見ただけで動きが読めて事前に対応できる」状態

成立条件：
  → 同じメンバーで長期間・同じ戦術を繰り返した経験
  → 相手チームの戦術が既知・固定的
  → 試合状況が訓練時と大きく乖離していない

暗黙知の限界（ハンドボール実戦）：
  → 相手が新しい戦術を使ってきたとき：即座に崩壊する
  → メンバー交代・怪我人発生時：暗黙知は引き継げない
  → 疲労・試合終盤：無意識の判断精度が下がる
  → 対戦未経験の相手：暗黙知は蓄積されていない
```

#### アイコンタクトとボディランゲージの限界

```
アイコンタクト（Blickkontakt）：
  有効距離：約5m以内（それ以上では表情・目線の読み取りが困難）
  問題点：
    → マーキング中は相手に正対しているため、
      味方とのアイコンタクトが物理的に取れないケースが多い
    → 高速の展開時（速攻・クイックスロー）は
      アイコンタクトする時間的余裕がない

ボディランゲージ（Körpersprache）：
  有効な場面：スクリーンのセット前・フリースロー前の静的状況
  問題点：
    → 動的な試合中（ドライブ対応・ローテーション）では
      周辺視野が低解像度のため読み取りが困難
    → 習慣化・標準化されていないジェスチャーは
      受け手が解釈できない
    → 国際大会・合同練習では動作の意味が共有されていない

結論：
  アイコンタクト・ボディランゲージは「補完的手段」に過ぎない
  声（Verbal Communication）は全距離・全動作状態で機能する
  唯一の高信頼性情報伝達手段である
```

### R-6. 集合知メカニズムのまとめ

```
DF6人の集合知 = 分散認知 × トランザクティブ・メモリー × リアルタイム声情報

  分散認知      → 6人で360°をカバーするシステム構造
  TMS          → 役割分担の暗黙的共有（誰が何を担当するかの確認省略）
  リアルタイム声 → 死角情報の即時補完（見えていないことを伝える）

声がなければ：
  → 分散認知は6つの孤立システムになる
  → TMSは機能するが死角情報が届かない
  → 結果：見えている範囲しか対応できない「6人の個人プレー」になる

声があることで：
  → 6人の視野が統合された「仮想的な全方位視野」が生まれる
  → TMSと組み合わさることで高速・省エネルギーな連携が実現する
  → 相手の戦術変化・予期しない動きにも対応できる柔軟性が生まれる
```
