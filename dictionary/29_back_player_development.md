## 【ドイツ DHB】 判断明確化 + バック-ピボット連携語彙

### AIM-Position（全動作可能受球姿勢）

**Alles-ist-möglich-Position**（アレス-イスト-メーグリッヒ-ポジション）＝ 「すべてが可能な姿勢」。

```
DHBが定義するバックプレーヤーの理想的なボール受球姿勢：

① 両膝を軽く曲げ、重心は均等（前後左右いずれにも即動けるニュートラル）
② 上体は正面向き（DF・GK・ピボットを同時に視野に入れる）
③ 利き手はシュート体勢に入れる位置、非利き手はパス方向を示さない
④ 足の向きでシュート方向を読まれないよう45°以内の角度制限

目的：ボールを持った瞬間に「シュート/ドライブ/パス」の3択が全て残っている。
    どれか1択が消えた状態でボールを受けると、DFはもう読んでいる。
```

**なぜ重要か（じゃんけん視点）**：
- AIM-Position の逆 = DF・GKに「読まれている」ポジション
- 上体が傾いている、利き手が先に動く、足がシュート方向を向く → これを見てDFは先読みする
- **バックプレーヤーの習慣チェック**：ボールを受けるとき、自分はAIM-Positionか？（→ 映像で確認 `28_video_analysis.md`）

### 4択判断ツリー（DHB Handball Praxis 15）

DHBの指導書「Handball Praxis 15」が定義する、バックプレーヤーが**ボールを持った瞬間に行う4択判断**。

> ⚠️ **重要：4択は「順番に考える」ものではない**
> 受球前の認知（`04_cognition.md`の3段構造）で**すでに答えを出しておく**。
> 下図はあくまで「4択の概念整理」。実際は全択を並列で評価し、ボールを持った瞬間には行動が決まっている状態が理想。

```
【4択の概念マップ】（受球前に並列評価する）

  ┌─────────────────────────────────┐
  │ [択1] 即シュート                  │
  │  スペースあり + GKが動く前に放てる  │
  ├─────────────────────────────────┤
  │ [択2] ドライブ                    │
  │  DFが近い + 1v1で勝てる角度がある  │
  ├─────────────────────────────────┤
  │ [択3] ピボットパス                 │
  │  DFが遠い + ポストへのコースが空    │
  ├─────────────────────────────────┤
  │ [択4] 横展開（リセット）            │
  │  択1〜3が全て潰されている          │
  │  → DFが動いたら択1〜3へ戻る       │
  └─────────────────────────────────┘
```

**使い方のポイント**：
- AIM-Position はこの「並列評価を最後まで保つ姿勢」として機能する
- 択4（横展開）は「逃げ」ではなく「じゃんけんをリセットして次の択を作る」判断
- 「受けてから考えた」試合は択4が多い。「受ける前に考えた」試合は択1〜3が多い

### バック-ピボット連携3概念（DHB標準語彙）

| 概念 | ドイツ語 | カタカナ | 意味 |
|---|---|---|---|
| シフト | Verlagern | フェアラゲルン | バックが動いてDFをずらす。ピボットのためのスペース創出 |
| スクリーン | Sperren | シュペレン | ピボットがバックのDFに対してスクリーンをかける |
| リリース | Absetzen | アプゼッツェン | スクリーン後にバックがDFから切り離れる動き |

```
連携の流れ：

  LB がフェアラゲルン（右へシフト）
      ↓
  DFが引きずられる
      ↓
  ピボットがシュペレン（LBのDFへスクリーン）
      ↓
  LBがアプゼッツェン（スクリーン裏に抜ける）
      ↓
  LBフリー → シュート or ピボットへリターンパス
```

**じゃんけん循環**：
- DFがシュペレンを嫌って早めにスイッチ → アプゼッツェンでピボットがフリー → ピボットシュート
- DFがスイッチしない → バック（LB）がアプゼッツェンでフリー → バックシュート
- どちらかが必ずフリーになる → ピック&ロールと同構造

## 【フランス FFHB】 フィクサシオン・デュ・ドゥミ・サントル

### Fixation du Demi-Centre（CBバイナリ判断ドリル）

**Fixation du Demi-Centre**（フィクサシオン・デュ・ドゥミ・サントル）＝ 「センターバックによる固定」。

> **概念の位置づけ**：「CBがDFのインターバル前に立つ」という考え方は多くの国が実践している普遍的な原則。フランスが独自なのは、**この動作を独立したドリルとして体系化・命名した点**。スペインでは「フィハシオン」、ドイツでは「Fixierung」と呼ぶ。

**コンセプト**：
CBが2人のDFの間隔の「正面」に立ち続けることで、DFに「左を切るか・右を切るか」の2択を強制する。

```
DF配置（例：5-1の後衛2枚）：

       [D_L]         [D_R]
              ↑
           ここにCBが立つ（インターバルの正面）

DFがとれる選択肢はバイナリになる：
  A：D_Lが前に出てCBを止める → D_Rの隣にスペース → RB/LWへパス
  B：D_Rが前に出てCBを止める → D_Lの隣にスペース → LB/RWへパス
  C：両者がプレッシャーしない → CBがシュート
```

**なぜこれが強力か**：
- DFが「どちらかが動かなければならない」状況を作り出す
- CBが**動かないこと**でDFを動かす（消極的能動性）
- 「そこにいるだけでDFが迷う」（`CLAUDE.md` 普遍原則5の例証）

**バックプレーヤーへの示唆（CBでない場合）**：
- LB/RBは「CBがインターバルに立っている」を確認してから動く
- CBが固定している間に、LB or RBはスペースを探してカットの準備
- 全員が動こうとするとCBの固定効果が消える

## 【デンマーク】 序盤20分コーディング + デュアルバック集中

### ギセル式「序盤20分コーディング」

デンマーク代表のミカエル・ハンセン等が実践、ラスムス・ランデルらが体系化した**試合序盤の情報収集フレームワーク**。

```
序盤20分でコーディングする4項目：

① DFポジショニング傾向
   → 自分の担当DFは「高い位置に出やすいか / 引いているか」
   → ボールサイドに寄りすぎる癖があるか

② GKの先読みパターン（→ 05_gk_df.md のGKタイプ判定と連動）
   → 3〜5本シュートで反応パターンを確認
   → 「何を見てから動くか」を観察

③ DF陣形の切り替えタイミング
   → 何点差になると守備陣形が変わるか（セーフティーDFに切り替えるポイント）
   → タイムアウト後に変わるか

④ ピボット対応DFの強度
   → ポストへのパスコースを積極的に消しているか
   → ポスト離れ気味に守るか
```

**コーディング後の活用**：
- 前半終了時点で「相手チームのじゃんけん法則」が1つ以上見える
- 後半最初の攻撃でそこを突く（相手が修正する前の「旬」）
- ハーフタイム分析（→ `28_video_analysis.md`）と連動させると精度が上がる

---

### デュアルバック集中モデル（50.2%データ）

IHF統計（2025年世界選手権）でデンマークの得点50.2%が**LB/RBの2名から**発生。

```
デュアルバック集中の構造：

  CBがオーガナイズ（パス配給・スペース管理）
      ↓
  LB / RB のどちらかに「解放する」判断
      ↓
  解放されたバックが「即シュートゾーン」に入る

  相手DFにとって：
  ・CBを警戒するとLB/RBが空く
  ・LB/RBを警戒するとCBが空く
  → じゃんけん循環（CBが管理、LB/RBが消費）
```

**バックプレーヤーへの示唆**：
- CBポジション：自分がシュートを打つよりも、LB/RBを「解放する」判断を優先する場面がある
- LB/RBポジション：CBが保持している時間は「シュートゾーンへの入り込み」に使う
- デュアルバックモデルでは**2枚目のバックがどれだけ積極的に動けるか**が鍵

## 【アイスランド】 マグヌッソン式 1v1特化訓練法

### 80-100回1v1 / セッション法

アイスランドのエリック・マグヌッソンコーチが確立した、**個人技術を戦術より先に完成させる哲学**。

```
訓練の原則：

  1セッション（90分）の構成：
  ・ウォームアップ：15分
  ・1v1ドリル集中：60分以上（80〜100回の反復）
  ・チーム練習：残り時間

  1v1ドリルの種類（すべてを繰り返す）：
  ① ステップフェイント（右/左 交互）
  ② ボディフェイク（上体/骨盤）
  ③ スピード変化（減速→加速）
  ④ ダブルフェイント
  ⑤ 接触を伴う1v1（DFが手を出す状況）
```

**なぜ80-100回が基準か**：
- 技術が**反射レベル（意識せずに出る）**になるためには数千回の反復が必要
- 1セッション80-100回 × 週4〜5回 × 数年 = 自動化のしきい値を超える
- チーム戦術はその後に乗せるもの（「土台なき家」を建てない）

**アイスランドが世界で通用する理由の核心**：
- 平均身長（短め）のハンデを個人技術の極限化で補う
- タクティクスを知っていても、1v1で負けると戦術は無意味
- 「技術が先、戦術は後」というフィロソフィー

**バックプレーヤーへの示唆**：
- 週あたりの1v1反復回数を数える（目安：1セッション50回以上が継続の基準）
- 試合に出た全フェイントを記録して「成功率の低いフェイント」を次セッションで多く練習
- 「苦手なフェイントを避ける」のではなく「苦手を繰り返す量で埋める」

> ⚠️ **レベル別注記**：80-100回はプロクラブ（全日練習、専任スタッフ）基準。
> **高校生の現実的目安：20〜40回/セッション**から始め、疲労なく精度が維持できる回数を基準にする。
> 量より「毎回フルコミット（本番と同じ強度）」で行うことが優先。

## 【クロアチア】 ジャンプシュート精度論 + フィート哲学

> **2つの理論の関係**：「ステップ数が少ない＝ジャンプシュートのエリート」と「立ちシュートが基本」は矛盾しない。
> ジャンプシュートを2〜3歩で打てるようになって初めて、「跳ぶ/跳ばない」の選択肢が生まれる。
> 最終形：**「2歩ジャンプ」と「立ちシュート」を同じ体勢から打ち分けられる状態** → DFもGKも読めない。

### ジャンプシュートバイオメカニクス：踏み切りステップ数理論

クロアチア代表の分析（EHF EURO 2022分析、Matkovićら）から導出された**踏み切りステップ数とエリートマーカーの相関**。

```
踏み切りステップ数と技術レベルの関係：

  ステップ多い（4〜5歩） → 助走距離を要する
    = まだシュートフォームが身体に馴染んでいない段階
    = DFが「助走開始を見て予測できる」

  ステップ少ない（2〜3歩） → 短い助走から最高点
    = フォームが自動化（意識せずに筋収縮できる）
    = DFが「どこから打つか読めない」
    → これがクロアチアのエリートバックの判定基準
```

**測定方法（自己分析）**：
- 自分のシュート映像を見て「踏み切り開始からジャンプまでのステップ数」を数える
- 同じシュートコースで3〜5本確認して平均を出す
- 疲労時・プレッシャー下で何ステップになるかも確認（ここが本当の技術レベル）

**なぜステップ数が減るか**：
- 体幹が安定し、踏み切り1〜2歩でも最高打点に到達できる
- 「助走から打点」ではなく「重心コントロールから打点」に変わる
- → 練習方法：立ち位置から1歩または2歩でのジャンプシュートを繰り返す

---

### 「フィート・オン・ザ・グラウンド」哲学

**"Keep Feet on the Ground" from 9m**（EHF EURO 2022 クロアチア分析）。

コンセプト：9m前後からのシュートは「必ずジャンプしなければいけない」わけではない。**立った状態からの直接シュートが基本**。

```
立ちシュート（スタントシュート）の優位性：

① DFが「跳ぶタイミング」で読んでいるところを裏切れる
   → DFとGKは「バックは跳ぶ」という経験値で待っている
   → 跳ばずに打つと読みのタイミングがずれる

② 打点の変化幅が大きい
   → 跳んでから打つ場合：最高点しか打点がない
   → 立ちシュート：どの高さからでも打てる（最高点・肩口・腰高）

③ 速い（フライトタイムが短い）
   → 助走+ジャンプの時間ゼロ
   → GKの先読みが間に合わない
```

**じゃんけん（ジャンプ vs 立ちシュート）**：
- DFがジャンプシュートに慣れて「跳んだら体を張る」 → 立ちシュートで裏をとる
- DFが「立ちシュートを警戒して前傾」 → ジャンプシュートで頭上を抜く
- これを試合内で1〜2回切り替えるだけで相手の読みを壊せる

## 【スペイン RFEBM】 スキャン-識別-コミット + 統一語彙

### ドゥシェバエフ「スキャン-識別-コミット」3ステップ

ダニ・ドゥシェバエフ（スペイン代表主将、ポルトガル戦術革命の中心）が言語化した**判断サイクル**。
スペイン代表のトレーニング映像分析から構成。

```
3ステップ判断ループ：

  [SCAN スキャン] ← ボールを受ける前0.5〜1秒
    ↓
    ・DFの数・位置・重心方向を素早く把握
    ・GKの構えを確認
    ・ピボットがどこにいるか

  [IDENTIFY 識別] ← ボールが手に来る瞬間
    ↓
    ・最大脅威はどのDFか
    ・最大チャンスはどのコースか
    ・「やっていい」か「やってはいけない」かのバイナリ判断

  [COMMIT コミット] ← ボールを受けた後即座
    ↓
    ・迷わない。「IDENTIFY」で出た答えを全力で実行
    ・中途半端は全て失敗（DFもGKも両対応できる）
```

**なぜ「コミット」が最重要か**：
- スキャンと識別が完璧でも、実行で迷うと全て無駄になる
- 迷い = DFに「もう一回見る時間」を与えること
- エリートバックプレーヤーの特徴：受球から行動開始まで**0.3秒以内**（→ 04_cognition.md）

**練習での習慣化**：
- 1v1練習中、「受ける前に声に出してスキャン結果を言う」（「右が空いている！」）
- コミットの確認：行動を起こした後「なぜそうしたか」を説明できるか
- できない = スキャンが機能していない（感覚だけで動いている）

**"Unified Handball Mind"（統一ハンドボール思考）**：スペイン連盟（RFEBM）が全国の指導者・選手に導入している**標準化された戦術語彙体系**。

| **Cruce** | クルセ | バック-バック間のクロス（クロスプレー全般） |
| **Cortina** | コルティーナ | スクリーン（DFの視野を遮る動作全般） |
| **Bloqueo** | ブロケオ | ブロック（身体で道を作る静的スクリーン） |
| **Desdoblamiento** | デスドブラミエント | 攻守切り替え後のポジション再分配 |

**なぜ統一語彙が重要か**：

- タイムアウト中に「コルティーナ→クルセ」と言えば全員が即座に動ける
- 言語が動作を媒介する → 思考速度が上がる

**日本のチームへの適用示唆**：
- チーム内で「スクリーン」「クロス」「固定」の定義をそろえる
- コーチが言った言葉と選手がイメージした動作が一致しているか確認する

## 【GK逆工学】 シューターがGKのキュー読みを逆用する

### GKキュー階層（PMC10591308）をシューターが逆から読む

GK研究論文（PMC10591308）が特定した**GKがシュート方向を予測するために読むキューの優先順位**。
シューター（バックプレーヤー）は「GKが何を見ているか」を知ることで、見せ方をコントロールできる。

```
GKがシュート前に読むキュー（優先順位順）：

  1位：上体回転（体幹）    ← GKが最も重視するキュー
  2位：手首の向き          ← リリース直前の方向決定キュー
  3位：助走経路           ← どの角度からアプローチするか
  4位：ブロックコンステレーション ← チーム全体の配置
```

**シューターが各キューを「偽る」方法**：

| GKが読むキュー | フェイクの方法 | 技術名 |
|---|---|---|
| **①上体回転** | 肩・胸を右に向けて実際は左へ打つ | ボディフェイク（上体フェイント）|
| **②手首** | 腕の振りは右方向、手首の返しで左コース | リストシュート |
| **③助走経路** | 右方向に走り込んで左ポストへ斜めシュート | クロスシュート |
| **④コンステレーション** | CBがフィクサシオン中、LBが逆サイドから無警戒ゾーンへ | ルーティング（陣形で空間を作る） |

**優先度の高いキューほど「偽る効果が大きい」**：
- ①上体回転を偽れると、GKは「行先が逆」と判断してしまう（最大効果）
- ②手首は難易度が高い（シュート精度が下がりやすい）
- ③助走経路はウィングが最も使う（バックはコースが限定されるため補助的）
- ④陣形全体でフェイクするのが最も持続可能（個人の体力に依存しない）

**GKの先読みを逆用する「ダブルブラフ」**：
```
通常：GKが上体フェイクを学習 → GK側は「上体フェイクの逆に動く」に変化
      ↓
  「上体フェイクをするとGKは逆に動く」を知っているシューター
      ↓
  上体フェイクなし（真正面）で本当に打つ → GKが「フェイク」と判断して反対に動く
      ↓
  これが「予測の予測を予測するスパイラル」（→ CLAUDE.md 普遍原則9）
```

## 【DF視点】このファイルをディフェンスで使う

> **前提**：このファイルの内容を知っていると「相手が何をしようとしているか」が読める。
> OF技術の裏にある「仕掛けの原理」を理解することが、最高のDFスカウティングになる。

### ポジション別DF活用

| 相手がやってくること | DFとして読むポイント | 対応 |
|---|---|---|
| **AIM-Position**（全オプション保持） | AIM-Positionが崩れた瞬間を狙う。体が傾いた・利き手が先に動いた = どちらかが消えた | そのタイミングで詰める。崩れる前は距離を保ってシュートブロックの準備だけ |
| **4択判断（択1〜4）** | 相手が「択4（横展開）」を選んだ = 現在の択1〜3が全部消えている = DFが機能している合図 | 横展開中に追わない。次の択が来るまで陣形を整える |
| **フィクサシオン（CB固定）** | 相手CBが動かない = LB/RBが動こうとしている = 見るべきは横の選手 | CBを追わず、解放されようとしているサイドに意識を向ける |
| **フェアラゲルン→シュペレン→アプゼッツェン** | バック選手が横に動いた = スクリーンが来る可能性 → 接触の前に「スイッチするか/粘るか」を決める | スクリーンを予測して先に動けると連携が崩れる |
| **序盤20分コーディング（相手が実施中）** | 序盤に相手バックが「試している」シュートを複数打つ = データ収集段階 | 序盤こそ「癖を見せない」。GKが右に動く癖があれば早めに修正を |
| **スキャン-識別-コミット** | 相手が「コミット前に一瞬止まる」タイミングがある = そこがDF介入のチャンス | 相手の視線が定まった瞬間に動く（コミットの逆を先取り） |
| **ステップ数少ないジャンプシュート** | ステップが少ない相手 = どこからでも打てる = 距離を詰めることで対応範囲が広がる | 助走開始を「見てから対応」は間に合わない → ポジションで先に潰す |
| **立ちシュート（フィート哲学）** | 跳ぶ動作がないため「ジャンプのタイミング」を読む方法が効かない | GKと連動して「立ちシュートのコース」を予め絞る（→ `05_gk_df.md` 解決策A） |

### DFとしてGK逆工学を使う

GKが読んでいるキュー（上体→手首→助走→陣形）を**DFが先につぶす**ことでGKを助けられる。

```
DFがGKのキュー読みを補助する方法：

  ①上体フェイクを使う相手     → DFがマークして上体を封じる（フェイク空間を消す）
  ②助走経路を変えてくる相手   → DFが立ち位置で経路を制限する（コースを1つにする）
  ③陣形全体でフェイクする相手  → DFが陣形を維持して「どこが空いているか」を変えない

→ GKが読むキューを「DFが予め消す」 = GKの負担を減らす = チームとしてのDF完成
```
