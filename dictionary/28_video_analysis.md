# 28. ビデオ分析・セルフスカウティング辞書

> **対象：** バックプレーヤーが自分の映像を分析して次のプレーを変える実践辞書。
> 「コーチが言うとバイアスがかかる。映像は嘘をつかない。」

---

## このファイルの哲学

このシステムの4軸フレームワークは試合中に「感覚」でやるが、映像を使えば試合後に「外の目」で検証できる。
映像分析は **③リアクション**（相手はどう動いたか）と **④自分の対応**（流れの中で自分は何をしたか）を客観化する唯一のツール。

```
感覚だけ → 主観バイアス（「うまくいった」「DFのせい」）
映像で検証 → 「受ける前に見ていなかった」「GKはコース絞り型だった」が分かる
言語化 → 次試合での再現・改善
```

## セルフスカウティング：バックプレーヤー分析チェックリスト

### シュート分析

```
□ 打った全シュートのコースを記録（左上/左下/中上/中下/右上/右下）
□ GKタイプを特定（先読み型 / コース絞り型 → 05_gk_df.md）
□ GKのタイプに合ったコース・タイミング選択ができていたか
□ 打点（最高点 / 早打ち / 踏み切りタイミング）は意図通りか
□ シュートフェイント後のGKの反応をコースに活かしたか
□ バックハンドフェイクを使ったか → 成否と理由
□ 7mスローのルーティンは実行できたか（→ 22_7m_throw.md）
```

### 認知・判断分析

```
□ ボールを受ける前の5ポイント確認（→ 17_scene_index.md Step 2）
  ① 自分のDFの位置・状態
  ② ピボットの位置
  ③ GKの位置
  ④ 隣の選手の状況
  ⑤ スペース（Tiefenraum）

□ ボールを受けてから行動までの時間（2秒以内か）
□ DFのポジション（クローズ/ワイド）を見てから行動できていたか（→ 07_formations.md）
□ ピボットへのパスルートを活用できたか
□ 「受ける前に考えていた」か「受けてから考えた」か
```

### フェイント・ドライブ分析

```
□ 使ったフェイントの種類（02_individual.md の7種を参照）
□ DFの注目部位（手/肩/骨盤）に合わせたフェイント選択か
□ ダブルフェイントの原則を使ったか
□ ドライブ突破の成否 → DFは後退/横/前のどれで対応したか
□ バックDF技術（Grundstellung/フットワーク）は維持できていたか（→ 23_back_df_individual.md）
```

## 映像の「どこ」を見るか

バックプレーヤーにとって試合中は「ボールを見ると全体が見えない」問題がある。映像は後から全体を見られる唯一の機会。

```
初級：ボールの動きを追う（誰がパスを受けたか）
中級：スペースを追う（空いているのはどこか）
上級：DFの重心を追う（DFが次にどこへ動こうとしているか）
最上級：「OF全体の意図」と「DF全体の対応」を同時に見る
```

**場面別の映像確認ポイント**：

| 場面 | 映像で確認すること |
|---|---|
| シュート場面 | 打つ前の「DFの位置」「GKの構え」を巻き戻して確認 |
| 失点場面 | 自分のポジションが適切だったか（映像なしでは確認不可） |
| 成功したプレー | なぜ成功したか言語化する（再現性のため） |
| ターンオーバー | 「自分の判断ミス」か「パス精度」かを切り分ける |
| 速攻場面 | 1stウェーブ・2ndウェーブの役割分担（→ 03_numerical.md） |

## タイムアウト・ハーフタイム用分析ワークフロー

### ハーフタイム15分でできること

```
1〜3分：映像確認（スマホでキーシーンを3つ以内）
4〜8分：選手・コーチで修正策を言語化（↓の優先順位で）
9〜14分：ウォームアップ
15分：再スタート
```

**修正優先順位**（映像から判断する順）：
1. 相手の得点パターン（何が通っているか）→ DFの穴
2. 自チームのターンオーバーパターン（どこでミスが出るか）
3. 得点チャンスを逃したシーン（判断の遅れか精度か）

## 分析ツール（無料〜低コスト）

| ツール | 主な用途 | 特徴 |
|---|---|---|
| **Coaching Eyes**（iOS/Android） | コマ送り・スロー再生・ライン描画 | 日本語対応・スマホで手軽 |
| **Hudl Technique** | スロー再生・サイドバイサイド比較 | トップコーチも使用 |
| **YouTube Studio** | 映像アップロードとコメント付け | 無料・チームで共有しやすい |
| **Dartfish Express** | 角度・速度測定 | 競技レベル向け |
| **標準カメラアプリ** | まず撮ることが最優先 | ハードルを下げる |

**撮影の注意点**：
- カメラは**コートの側面かつ高い位置**（全体が見えるように）
- バックプレーヤー1人にフォーカスした映像と全体映像の両方があると理想
- 最低でも**自分の全プレーシーン**を記録（試合全部でなくてよい）

## じゃんけん（映像分析）

```
映像を見ない状態（感覚だけ）
  → 主観バイアス（「うまくいった」「DFのせい」「運が悪かった」）
  → 映像で客観化 → 「受ける前に見ていなかった」が分かる
  → 意識化 → 次試合で改善
  → 新しい盲点が生まれる → また映像で客観化 → ループ

相手スカウティング（映像あり）
  → 相手も自チームをスカウティングして対策を立てる
  → サインプレーの変更・GKのコース対策・DF陣形の修正
  → こちらも再スカウティング → じゃんけんの軍拡競争

「見られているとわかると人間は行動を変える」
  → 意図的に"罠"のプレー（わざと同じパターン）でスカウティングを誘う
  → 実際は別のパターンで攻める → サインプレーの偽装
```
