# ハンドボール自己問答システム — プロジェクト文脈ファイル

このファイルを読めば、どのPC・どのセッションからでも作業を継続できる。

---

## プロジェクト概要

**目的：** 高校ハンドボール部（全国レベル）のバックプレーヤーが試合後に自分で振り返り、次の行動を自分で決める「自己問答システム」。

**コーチ（東岸さん）の哲学：**
- 「この世にある全てのハンドボールを私は認めている。相性や向き不向きがあるだけ」
- じゃんけん構造：絶対最強の戦術は存在しない
- OF↔DFは相互依存：「DFに合うOF、それに合うDF……」という連鎖
- バイアスなき辞書：コーチが言うとバイアスがかかるので、AIが全羅列する

**対象選手：** バックプレーヤー（ウィングではない）

---

## システムの2つの役割

1. **ハンドボール辞書**：全戦術・技術・状況を羅列・パターン分け・じゃんけん関係を整理
2. **伴走者**：個別最適化ガイド（選手の強み×現在の課題→具体的アドバイス）

---

## 4軸質問フレームワーク（アプリの核心構造）

```
①自分の立場    → 今回がOF側かDF側か
②相手の状況    → 相手はどんな選択肢を取ろうとしていたか（認知）
③リアクション理解 → 自分の行動に対して相手はどう動いたか
④自分の対応    → ④までの流れの中で自分は何をしたか

出力：良かったところ ／ 課題点 ／ 改善案
```

## コーチ補足知識（必ずコードに反映すること）

### GK
- **3方向の法則**：左/中/右（常に2/3が閉じていて1/3が開いている）
- **波型GK**：コース予測で先に動く → 早打ちが有効（GKが動く前に打つ）
- **スポット型GK**：動かずにコースを絞る → 引きつけてから最後に角度で打つ
- GK×DF連携：DFがコースを絞ってGKの仕事を減らす

### ポストの4プレー
1. **回転プレー**：DFに背中を向けた状態から体を回転させてシュート
2. **フェイクプレー**：受けた瞬間に逆方向フェイクでDFを外す
3. **フラッシュプレー**：6mライン付近に瞬間的に飛び込んでパスを受ける
4. **スクリーンプレー**：ポストがDFに壁を作り、バックプレーヤーの突破を助ける

### コンタクト技術（山側/谷側）
- DFが斜めに構えたとき「山側（高い方）」を越えさせないのが原則
- 「谷側（低い方）」は前後ステップで押し出す
- コンタクト中も常に次のポジション（勝ち位置）を意識

### ポストDF V字位置取り
- ポストOFとゴールを結ぶ線上（V字の頂点）に立つ
- ポストOFより先にボールを受ける位置を取る（先読み）

## 普遍原則（全辞書を貫く）

1. **先に読んだ方が選択肢を持てる**
2. **2/3閉じて1/3開ける**（GKの法則→DF全体に転用）
3. **じゃんけん構造**（絶対最強なし、相性と文脈のみ）
4. **認知→アクション→リアクション**（全プレーの3段構造）
5. **ポストの核心**：「そこにいるだけでDFが迷う」
6. **個人特性が戦術を決める**
7. **試合文脈が全てを上書きする**
