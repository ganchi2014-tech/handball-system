# 届きやすさテスト（Findability Test）

**目的**：日本の高校生が「知りたい状況」に置かれた時、現アプリで何秒/何クリックで該当情報に到達できるかを実測する。
**設計哲学**：「知りたい時に手に届く」ジャストインタイム設計を検証する。
**実施日**：2026-05-24

## テスト方法

各シナリオで以下4経路を試す：
- **P1：辞書 → キーワード検索**（平易語で）
- **P2：辞書 → タグフィルタ**
- **P3：診断（課題解決） → カテゴリ→症状**
- **P4：振り返り → モード選択**

各経路の **到達秒数・クリック数** を測定。

## 評価基準

| 結果 | 意味 |
|---|---|
| ✓ 到達（≤3クリック） | 「届く」設計が機能 |
| △ 到達（4〜6クリック） | 改善余地あり |
| ❌ 到達不能 or 7+クリック | 「届かない」＝修正必須 |

### 主要発見

1. **ユニーク用語（シューズ／松ヤニ）→ 完璧に届く**：新辞書31が独占的にヒット
2. **一般用語（ファウル／パッシブ／退場）→ 埋もれる**：既存辞書（特に00じゃんけんマップ）がトップを占め、新30_rulesが見えない
3. **「振り返り」「切り替え」**：抽象語ほど 00 が独占、特化辞書に届きにくい

### 根本原因

- 検索結果ソートが「タイトル一致 > 本文一致」になっていない
- 00_janken_mapは横断統合インデックスのため、ほぼ全キーワードでヒットする宿命
- **検索結果のランキングロジック改善が必要**

### 改善方針

| 方針 | 効果 | 実装難度 |
|---|---|---|
| A. ファイルID降順ソート | 新辞書が先に出る | 低 |
| B. タイトル一致＞本文一致の優先 | 関連度高い辞書が先 | 中 |
| C. **P3経路を充実（課題解決カテゴリ追加）** | 症状→処方箋で直接案内 | 中（既存パターン） |
| D. 「キーワードランディング辞書」追加 | "ファウル"等で必ず30_rulesに飛ぶリダイレクト | 高 |

→ **Phase 3-B として方針C（課題解決カテゴリ追加）を実装。設計哲学（症状ベースのジャストインタイム）と最も整合**。

### Phase 3-B 計画

SOLVE_DATA.player.categories に以下3カテゴリ追加：
1. **rules**：ルール・反則で困っている（5症状）→ 30_rules へ誘導
2. **equipment**：装備・道具で迷っている（5症状）→ 31_equipment へ誘導
3. **matchday**：試合当日の動きで不安（5症状）→ 32_match_day へ誘導
